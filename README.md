## PSY MI — React + Vite + Express Server

Полная инструкция по запуску проекта с регистрацией и личным кабинетом.

---

## 📋 Предварительные требования

1. **Node.js** (версия 18 или выше)
2. **PostgreSQL** (версия 12 или выше)
3. **npm** или **yarn**

---

## 🚀 Пошаговая инструкция по запуску

### Шаг 1: Установка зависимостей

Откройте терминал в корневой папке проекта и выполните:

```bash
# Установка зависимостей для фронтенда
npm install

# Установка зависимостей для сервера
cd server
npm install
cd ..
```

### Шаг 2: Настройка базы данных PostgreSQL

1. **Установите PostgreSQL** (если еще не установлен):
   - macOS: `brew install postgresql@14` или скачайте с [postgresql.org](https://www.postgresql.org/download/)
   - Windows: скачайте установщик с официального сайта
   - Linux: `sudo apt-get install postgresql` (Ubuntu/Debian)

2. **Создайте базу данных**:
   ```bash
   # Войдите в PostgreSQL
   psql -U postgres
   
   # Создайте базу данных
   CREATE DATABASE psy_mi;
   
   # Выйдите из PostgreSQL
   \q
   ```

### Шаг 3: Настройка переменных окружения

#### 3.1. Создайте файл `.env` в папке `server/`:

```bash
cd server
touch .env
```

Откройте файл `server/.env` и добавьте следующие переменные:

```env
# База данных PostgreSQL
DATABASE_URL="postgresql://ваш_пользователь:ваш_пароль@localhost:5432/psy_mi?schema=public"

# Секретный ключ для JWT токенов (минимум 32 символа)
JWT_SECRET="ваш-случайный-секретный-ключ-минимум-32-символа-длинный"

# Порт сервера (по умолчанию 3000)
PORT=3000

# URL фронтенда (для восстановления пароля)
FRONTEND_URL="http://localhost:5173"

# Настройки SMTP для отправки email (опционально, для восстановления пароля)
# Если не настроены, токены будут выводиться в консоль в режиме разработки
SMTP_HOST="smtp.gmail.com"
SMTP_PORT="587"
SMTP_USER="ваш-email@gmail.com"
SMTP_PASS="ваш-пароль-приложения"
SMTP_FROM="noreply@psymi.com"
```

**Важно:** Замените:
- `ваш_пользователь` и `ваш_пароль` на ваши учетные данные PostgreSQL
- `ваш-случайный-секретный-ключ...` на случайную строку минимум 32 символа
- SMTP настройки (если хотите отправлять email)

#### 3.2. Создайте файл `.env` в корне проекта (для фронтенда):

```bash
cd ..
touch .env
```

Откройте файл `.env` в корне и добавьте:

```env
# URL API сервера
VITE_API_URL=http://localhost:3000/api
```

### Шаг 4: Генерация Prisma клиента

```bash
# Из корневой папки проекта
cd server
npm run prisma:generate
cd ..
```

### Шаг 5: Применение миграций базы данных

```bash
# Из корневой папки проекта
cd server
npm run prisma:migrate
cd ..
```

Если это первый запуск, Prisma создаст все необходимые таблицы в базе данных.

### Шаг 6: Запуск проекта

**Вариант 1: Запуск фронтенда и сервера одновременно (рекомендуется)**

Из корневой папки проекта выполните:

```bash
npm run dev
```

Эта команда запустит:
- **Сервер** на `http://localhost:3000`
- **Фронтенд** на `http://localhost:5173`

Вы увидите в терминале два окна:
- `[SERVER]` — логи сервера (синий цвет)
- `[CLIENT]` — логи фронтенда (малиновый цвет)

**Вариант 2: Запуск отдельно**

Если нужно запустить отдельно:

```bash
# Терминал 1: Запуск сервера
cd server
npm run dev

# Терминал 2: Запуск фронтенда
npm run dev:client
```

## 🐛 Решение проблем

### Ошибка: "Prisma client did not initialize yet"

```bash
cd server
npm run prisma:generate
cd ..
```

### Ошибка: "Cannot connect to database"

1. Проверьте, что PostgreSQL запущен:
   ```bash
   # macOS/Linux
   brew services list
   # или
   sudo service postgresql status
   ```

2. Проверьте правильность `DATABASE_URL` в `server/.env`

3. Убедитесь, что база данных `psy_mi` создана

### Ошибка: "Port 3000 is already in use"

Измените порт в `server/.env`:
```env
PORT=3001
```

И обновите `VITE_API_URL` в корневом `.env`:
```env
VITE_API_URL=http://localhost:3001/api
```

### Сервер не запускается

1. Убедитесь, что все зависимости установлены:
   ```bash
   npm install
   cd server && npm install && cd ..
   ```

2. Проверьте, что Prisma клиент сгенерирован:
   ```bash
   cd server && npm run prisma:generate && cd ..
   ```

## 📝 Что установится

**Фронтенд:**
- react, react-dom — UI
- react-router-dom — маршруты
- react-hook-form — формы
- zod, @hookform/resolvers — валидация
- axios — HTTP-клиент
- zustand — стейт-менеджмент
- clsx — удобная склейка классов

**Бэкенд:**
- express — веб-сервер
- prisma — ORM для работы с БД
- bcryptjs — хеширование паролей
- jsonwebtoken — JWT токены
- nodemailer — отправка email
- cors — CORS поддержка

**Dev-инструменты:**
- typescript — типы
- vite — сборка/дев-сервер
- eslint + prettier — линтинг и форматирование
- concurrently — одновременный запуск серверов
